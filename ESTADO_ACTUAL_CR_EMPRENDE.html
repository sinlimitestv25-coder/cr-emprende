Resumen general

La plataforma ya cuenta con flujo de administrador, usuarios reales, demos por tiempo, panel comercial, rubros editables, finanzas, historial de alertas, exhibición pública por emprendimiento, consultas desde portal y registro manual de ventas con comisión.

React + Vite LocalStorage Panel admin Portal público Comisiones manuales

Base del proyecto

Aplicación El proyecto funciona como una aplicación React con Vite. El archivo principal de comportamiento está en src/App.jsx y los estilos generales en src/index.css.
Persistencia actual La información se guarda en localStorage. Esto sirve para prototipo y prueba local. Para producción real conviene pasar a base de datos, autenticación y almacenamiento de imágenes.

Login y acceso

  • El administrador usa el correo reservado admin@cremprende.com.
  • Los usuarios creados ingresan con correo @cremprende.com y contraseña temporal.
  • Los usuarios demo tienen duración por minutos u horas: 5 minutos, 1 hora, 5 horas, 12 horas o 24 horas.
  • El sistema muestra un reloj de cuenta regresiva durante la demo.
  • Cuando vence la demo, se cierra la sesión y aparece un mensaje final con acciones de WhatsApp y cierre.

Panel administrador

Dashboard Incluye tarjetas de resumen, estados de usuarios, gráfico de usuarios por rubro y alertas comerciales útiles.
Alertas Las alertas priorizan pendientes de pago y usuarios sin emprendimiento creado. Las acciones se registran en historial comercial/financiero.

Usuarios

  • Se pueden crear usuarios normales y usuarios demo.
  • El correo se genera con el nombre y dominio @cremprende.com.
  • Se puede definir contraseña temporal, rubro, plan, estado de pago y duración de demo.
  • El usuario creado ya puede ingresar con sus datos reales.

Emprendimientos

  • Se quitaron accesos directos del administrador al panel del usuario.
  • El ingreso al panel del emprendedor queda sujeto a permiso del usuario.
  • El buscador permite localizar por nombre, usuario/responsable y rubro.
  • Se ajustó la terminología para evitar usar "dueño" como etiqueta principal.

Rubros

Gestión editable Cada rubro puede editar nombre, enfoque, etiqueta, ejemplos, actividades y módulos asociados.
Expansión visual Al seleccionar un rubro, se expande en el centro ocupando el espacio principal. Al volver a seleccionarlo, regresa a su tamaño normal.

Finanzas

  • Se mantiene el registro de ingresos y horas.
  • Se agregó historial de seguimiento comercial.
  • Se registran acciones provenientes de alertas.
  • Se incorporó configuración de comisión del portal por emprendimiento.
  • La comisión contempla porcentaje y tope mensual manual.

Exhibición y portal público

Módulo Exhibición El usuario puede crear publicaciones con título, descripción, precio opcional, imagen por URL o archivo local, y estado visible.
Portal público Cada emprendimiento tiene un link público para compartir por WhatsApp, estados o redes. El visitante puede ver productos y enviar consultas.
Personalización El usuario puede editar descripción del portal, color principal y usar identidad visual del emprendimiento junto al logo de Emprende.
Consultas El visitante deja nombre, WhatsApp y mensaje. La consulta queda registrada para que el emprendedor pueda responder y gestionarla.

Clientes y oportunidades

  • Una consulta del portal puede pasar a estado potencial.
  • Los potenciales del portal se reflejan en el módulo de clientes.
  • Cuando la venta se concreta, el usuario carga el monto vendido.
  • El sistema calcula la comisión según el porcentaje y el tope mensual configurados.

Comisiones del portal

Elemento Funcionamiento actual
Porcentaje Lo define el administrador desde Finanzas.
Tope mensual Limita la comisión máxima que se cobra en el mes por emprendimiento.
Registro La venta queda registrada en finanzas y visible para seguimiento.
Aviso al usuario Si cambia la configuración de comisión, el usuario ve un aviso al ingresar y puede marcarlo como entendido.
Pago Por ahora queda manual. No hay integración automática con Mercado Pago, Stripe ni transferencia bancaria.

Pendientes recomendados

  • Migrar datos de localStorage a una base de datos real.
  • Agregar autenticación segura para administrador y usuarios.
  • Guardar imágenes del portal en storage real.
  • Definir política comercial final de comisiones y topes.
  • Agregar exportes o reportes mensuales para finanzas.
  • Preparar dominio y publicación web definitiva.
